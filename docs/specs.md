## 产品规格说明书（MVP）

**产品名称**：WriteNow  
**产品定位**：纯网页版、免登录、极简 Markdown 转公众号排版工具  
**Slogan**：Write, Format, Publish.  
**版本**：MVP v1.0  
**文档日期**：2026-04-13

---

### 一、产品概述

#### 1.1 产品背景
- 技术创作者、产品经理、独立开发者习惯用 Markdown 写作，但微信公众号后台不原生支持。
- 现有排版工具（壹伴、135编辑器）功能臃肿，需安装插件或登录，学习成本高。
- 开源方案（doocs/md、mdnice）功能强大，但界面复杂，缺少审美元气与极简体验。

#### 1.2 产品目标
打造一款 **“打开即用、复制即走”** 的公众号排版工具，让用户在 **30 秒内** 完成从 Markdown 到精美排版的全流程。

#### 1.3 核心差异化
| 维度 | 竞品 | WriteNow |
|:---|:---|:---|
| 账号体系 | 需登录 | **完全免登录** |
| 功能定位 | 大而全 | **只做 Markdown 转公众号** |
| 审美体验 | 通用模板 | **3套设计师打磨的主题 + 呼吸感设计** |
| 上手时间 | 5-10分钟 | **小于30秒** |
| 交互细节 | 功能堆叠 | **聚焦核心任务，微动效精致反馈** |

### 四、技术方案

#### 4.1 整体架构
纯前端 SPA，无后端，静态托管。

```
用户浏览器
    │
    ├── Markdown 编辑器（CodeMirror 6）
    ├── 解析引擎（markdown-it）
    ├── 样式内联引擎（juice 轻量版）
    └── 剪贴板 API
```

#### 4.2 技术选型

| 层面 | 技术栈 | 选型理由 |
|:---|:---|:---|
| **前端框架** | React 18 + TypeScript | 生态成熟，组件化效率高 |
| **Markdown 编辑器** | CodeMirror 6 | 轻量、可定制、语法高亮 |
| **Markdown 解析** | markdown-it | 插件化架构，易于扩展 |
| **代码高亮** | highlight.js | 支持190+语言，主题丰富 |
| **样式内联** | juice（浏览器轻量版） | 成熟 CSS 内联库 |
| **剪贴板操作** | navigator.clipboard API | 现代浏览器标准 API |
| **部署** | Vercel / Cloudflare Pages | 免费、全球 CDN、自动部署 |

#### 4.3 核心流程

```
[用户输入 Markdown] 
       ↓
[markdown-it 解析为 HTML] 
       ↓
[应用当前主题 CSS] 
       ↓
[highlight.js 高亮代码块] 
       ↓
[juice 将 CSS 内联为 style 属性] 
       ↓
[生成微信公众号安全 HTML] 
       ↓
[写入剪贴板（text/html + text/plain）]
```

#### 4.4 样式内联化微信兼容处理
- 将 `<div>` 替换为 `<p>` 或 `<section>`
- 移除微信不支持的 CSS（如 `display: flex`）
- 压缩 `style` 属性空格与换行
- 仅使用行内样式，无 `<style>` 标签与 `class`

#### 5.1 布局结构

```
┌───────────────────────────────────────────────────────┐
│  [WriteNow]                     [极客代码] [文艺] [商务]  │
├────────────────────────────┬──────────────────────────┤

│       Markdown 编辑器       │      │  手机线框    │     │

└───────────────────────────────────────────────────────┘
```

#### 5.2 体验增强细节

| 交互点 | 具体实现 |
|:---|:---|
| **默认示例** | 页面加载后编辑器展示一篇完整示例文章，预览区同步渲染 |
| **粘贴即排版** | 监听 `paste` 事件，若光标在编辑器外且内容疑似 MD，自动替换编辑器内容并提示“已识别 Markdown” |
| **主题切换** | 预览区内容 `opacity` 淡出→停顿0.1s→新样式淡入，过渡 0.2s |
| **复制反馈** | 点击“复制到公众号”后，按钮文字变为“✅ 已复制！”，伴有轻微波纹动画，2秒后恢复 |
| **编辑器焦点** | 聚焦时左侧边框出现主题色微光效果 |
| **语法提示条** | 编辑器为空时底部显示 `**加粗** *斜体* [链接](url) ```代码``` `，打字后淡出 |
| **代码语言标签** | 识别 ` ```js ` 等，预览区代码块右上角显示半透明 `JavaScript` 标签 |
| **手机框装饰** | 预览区外包裹微圆角细边框，顶部可选加极小“刘海”装饰线 |

#### 5.3 设计系统

| 要素 | 规范 |
|:---|:---|
| **配色** | 主背景纯白/极浅灰（`#FAFAFA`），主按钮强调色 `#1A5F7A`（深蓝绿） |
| **字体** | 编辑器 `JetBrains Mono`, `Fira Code`；预览/界面 `-apple-system, BlinkMacSystemFont` |
| **圆角** | 按钮、卡片、预览框统一 `12px-16px` |
| **阴影** | 弥散阴影 `0 10px 30px -10px rgba(0,0,0,0.05)` |
| **内边距** | 编辑器内边距 `24px`，行高 `1.6` |
| **加载态** | 骨架屏（Skeleton Screen）展示主要布局结构 |

### 八、风险与应对

| 风险点 | 应对措施 |
|:---|:---|
| 样式内联化在微信后台失效 | MVP 上线前在真实公众号后台多轮测试；提供纯文本降级方案 |
| 代码块在手机端溢出 | 主题中为 `pre` 添加 `overflow-x: auto` |
| 图片依赖外部链接被微信过滤 | MVP 不处理图片上传，引导用户手动通过公众号后台上传 |
| 竞品快速跟进体验细节 | 持续迭代主题审美与动效细节，建立“设计感”心智壁垒 |
