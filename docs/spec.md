## 产品规格说明书（Product Spec）

**产品名称（暂定）**：即刻排版 · 公众号极简助手  
**版本**：MVP v1.0  
**文档版本**：1.0  
**更新日期**：2026-04-11

---

### 1. 产品背景与目标

**市场背景**：
- 壹伴、135编辑器等头部产品功能日益臃肿，学习成本上升
- 壹伴网页版存在素材导入故障、浏览器兼容性差、强制登录等体验痛点
- 大量个人博主、学生、轻量创作者只需要基础排版，不需要复杂的数据分析或AI写作

**产品目标**：
- 做一款**真正的“打开即用”**的公众号排版工具
- **免登录**完成全部排版流程，降低使用门槛
- 专注解决“格式错乱”和“移动端预览”两大核心痛点
- 提供比壹伴网页版更稳定、更轻量的纯网页体验

---

### 2. 用户画像

| 用户类型 | 核心需求 | 当前痛点 |
|----------|---------|----------|
| **个人生活博主** | 简单排版，快速发文 | 不想花钱，不想学复杂功能 |
| **大学生/实习生** | 完成社团/作业推文 | 学校电脑限制安装插件 |
| **技术写作者** | 代码块美观，排版省时 | 代码高亮在公众号里很难看 |

| **偶尔发文的职场人** | 不常发，不想记账号密码 | 注册登录流程烦琐 |

### 3. 核心差异化（与壹伴网页版对比）

| 对比维度 | 壹伴网页版 | 即刻排版 |
|----------|-----------|---------|
| 账号体系 | **必须登录** | **完全免登录**使用（保存记录可选登录） |
| 功能定位 | 插件版补充，功能大而全 | **网页优先**，只做排版这一件事 |
| 付费策略 | 免费版功能严重受限 | **核心功能永久免费**，仅增值功能收费 |
| 稳定性 | 素材导入易故障，字体调整偶发失效 | **全浏览器兼容**，核心功能100%稳定 |
| 上手时间 | 约10分钟熟悉界面 | **小于30秒**即可完成首次排版 |

### 4. 功能模块详设（MVP版）

#### 4.1 核心排版引擎（必做）
- **富文本编辑器**：基于 `Slate.js` 或 `Quill.js` 定制，支持：
  - 字号（12px-24px）
  - 字体颜色/背景色
  - 对齐方式（左/中/右/两端）
  - 行距（1.5/1.75/2.0/自定义）
  - 字间距、页边距调整
- **格式清洗**：粘贴Word/网页内容时，自动清除冗余样式，只保留纯文本与基础段落结构
- **图片处理**：支持拖拽/粘贴上传，自动压缩至公众号建议尺寸（需后端配合或使用浏览器压缩API）

#### 4.2 一键排版与模板（必做）
- **模板库**：首期上线 **10套** 设计精良的基础模板，覆盖：
  - 文艺清新（适合生活随笔）
  - 科技极简（适合技术教程）
  - 商务大气（适合职场干货）
  - 深夜电台（适合情感文章）
- **智能识别排版**：识别文章内的加粗、引用、列表，自动匹配对应模板样式
- **样式内联化**：一键导出时，将所有CSS样式转换为行内样式（`style=""`），确保粘贴到公众号后台 **100%不丢失格式**

#### 4.3 内容类型驱动推荐（必做 · 差异化亮点）
- **文章类型识别**：用户在编辑器中粘贴文章后，点击“智能推荐”，系统通过标题和正文关键词（调用低成本NLP API或前端规则引擎）判断文章类型：
  - `技术教程` → 推荐代码高亮模板 + 步骤卡片样式
  - `产品测评` → 推荐图文穿插样式 + 对比表格样式
  - `情感故事` → 推荐文艺字体 + 留白呼吸感样式
- **结果展示**：顶部弹出轻提示：“检测到这是一篇技术教程，已为您推荐【极客代码】风格”，一键确认即可套用

#### 4.4 移动端实时预览（必做）
- 右侧固定预览窗口，尺寸锁定为 **375x667**（模拟手机屏幕）
- 支持点击展开全屏预览二维码（方便手机扫码实测）
- 注意：**免登录场景下，预览图生成完全在浏览器本地计算**，无需上传服务器

#### 4.5 核心交互流（免登录闭环）
1. 用户访问域名 `jikepaiban.com`
2. 直接进入编辑界面，无弹窗，无登录浮层
3. 点击“示例文章”可快速体验（内置3篇不同风格的示例文案）
4. 编辑满意后，点击顶部大按钮“**复制到公众号**”
5. 系统处理样式内联化，复制富文本到剪贴板
6. 提示“复制成功，请到公众号后台粘贴”，同时右下角弹出轻提示：“登录后可保存本次排版记录”

### 5. 非功能需求（体验保障）

| 需求项 | 具体要求 |
|--------|---------|
| **浏览器兼容** | Chrome 90+， Edge 90+， Firefox 100+， Safari 15+ |

| **海外访问** | 静态资源部署Vercel/Cloudflare全球CDN，确保跨国访问速度 |
| **隐私保护** | 免登录模式下，文章内容不上传服务器，100%本地处理 |

### 6. 技术栈建议

| 层面 | 技术选型 | 理由 |
|------|---------|------|
| **前端框架** | React 18 + TypeScript | 生态成熟，组件化开发效率高 |
| **编辑器核心** | Slate.js | 定制能力强，对复杂样式控制力优于Quill |
| **样式内联** | juice（Node库）或自研CSS解析 | 将`<style>`标签转为行内样式 |
| **部署与存储** | Vercel（前端托管）+ 可选Cloudflare Workers | 免费额度足够支撑MVP流量，且全球加速 |
| **NLP识别（可选）** | 前端关键词规则库 + 百度/阿里免费NLP额度 | 控制成本，先用规则，后期再上模型 |

### 8. 商业模式构想（不干扰免费体验）

- **Freemium模型**：
  - 免费用户：无限次使用全部基础排版功能 + 10套标准模板
  - Pro用户（￥15/月或￥99/年）：
    - 解锁全部高级模板（每月新增）
    - 云端保存排版历史（最多50篇）
    - 无水印高清封面图生成工具
    - 批量图片压缩与上传

- **关键原则**：**绝不因未付费而强制加水印或限制复制功能**。唯一的“隐形广告”是在复制成功后的轻提示里推荐Pro功能。

### 9. 风险提示与应对

| 风险点 | 应对策略 |
|--------|---------|
| **壹伴更新网页版体验** | 坚持极简定位，不做功能堆砌；快速迭代建立用户心智 |
| **样式内联技术不稳定** | 预留降级方案：若自动转换失败，提供“纯文本+样式代码”手动模式 |
| **公众号官方限制外链图片** | MVP阶段不依赖图床，引导用户手动上传图片（排版工具仅负责样式） |
