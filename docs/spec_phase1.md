### Phase 1 Spec：成文之基（MVP）

**版本目标**  
搭建产品基石：用户能注册登录，云端保存文章，导入 Markdown/Word/WPS/Notion，并能对单篇文章进行排版，适配公众号、小红书等平台预览和一键复制。

**功能列表与详述**

| 功能模块 | 需求编号 | 功能描述 | 优先级 |
|---|---|---|---|
| 用户系统 | P1-01 | **邮箱注册/登录**：支持邮箱+验证码或邮箱+密码注册登录。包含密码重置流程。 | P0 |
| | P1-02 | **微信扫码登录**：支持微信 OAuth 2.0 登录，快速绑定/注册账号。 | P1 |
| | P1-03 | **账户设置**：修改头像、昵称、密码。注销账户。 | P2 |
| 云端保存 | P1-04 | **自动保存/同步**：编辑时每隔30秒自动保存到云端，多设备登录后拉取最新内容。网络异常时本地缓存。 | P0 |
| | P1-05 | **草稿箱管理**：文章列表，支持新建、删除、重命名、搜索。按修改时间排序。 | P0 |
| 多格式导入 | P1-06 | **导入 Markdown 文件**：上传 .md 文件，解析为标准 Markdown 并转存到内置编辑器。 | P0 |
| | P1-07 | **导入 Word 文件**：上传 .docx，解析文本、图片、基础样式（粗体、斜体、标题层级）为可编辑格式。 | P1 |
| | P1-08 | **导入 WPS 文件**：复用 Word 导入逻辑，同时兼容 .wps 格式（需转换或解析库）。 | P2 |
| | P1-09 | **导入 Notion 导出文件**：支持导入 Notion 导出的 Markdown+CSV 或 HTML 格式，保留页面结构和数据库。 | P2 |
| | P1-10 | **粘贴导入**：直接粘贴富文本/Markdown 至编辑器，自动解析为统一格式。 | P1 |
| 排版引擎 | P1-11 | **富文本编辑器**：基于开源编辑器（如 ProseMirror/TipTap）实现，支持标题、段落、加粗、斜体、引用、列表、链接、图片、分割线等。 | P0 |
| | P1-12 | **多平台样式预设**：内置“公众号”、“小红书”、“知乎”、“通用长文”等多种视觉风格模板，一键切换，即时预览效果。模板定义字体大小、行距、段距、配色、字色。 | P0 |
| | P1-13 | **自定义样式**：用户可修改文字颜色、背景色、边距，可添加自定义 CSS 片段。 | P1 |
| | P1-14 | **平台预览**：提供 phone 框架预览图，模拟小红书/公众号实际显示效果。 | P1 |
| 导出与分发 | P1-15 | **一键复制富文本**：将排版后的内容以兼容公众号等平台的富文本形式复制到剪贴板。 | P0 |
| | P1-16 | **导出文件**：支持导出为 Markdown、HTML、PDF。 | P1 |

**关键用户流程**  
1. 用户注册/登录 → 进入文稿列表。  
2. 新建文章 → 在编辑器中写作，或直接导入 Markdown/Word 文件 → 系统自动保存。  
3. 点击“排版” → 选择“公众号”风格 → 实时预览并微调 → 一键复制 → 粘贴到公众号后台。  
4. 退出后重新登录，草稿仍在列表，可继续编辑或导出。

**数据模型要点**  
- **User**：id, email, password_hash, nickname, avatar, wechat_openid, created_at  
- **Article**：id, user_id, title, content (JSON格式的 ProseMirror 文档), plain_text, word_count, platform_style, import_source, status (draft/published), created_at, updated_at  
- 支持版本快照：ArticleVersion (id, article_id, content, created_at) 用于回溯。

**技术考量**  
- 编辑器选型：ProseMirror 或 Tiptap，可灵活输出 HTML/Markdown，支持协作占位。  
- 文件导入解析：用 mammoth 库解析 docx，用 marked 解析 md，WPS 可尝试后台转换。  
- 自动保存：前端定时发送差分或全量内容到后端，用 IndexedDB 做离线缓存。  
- 复制富文本：需生成带内联样式的 HTML 并写入剪贴板，注意图片处理需上传图床转 URL。

**非功能性需求**  
- 支持 Chrome/Safari/Firefox/Edge 最新两个版本。  
- 编辑器首次内容绘制 < 1s，自动保存间隔无感知。  
- 接口 QPS 支撑日常并发，支持横向扩展。

### Phase 2 Spec：灵感成文

**版本目标**  
连接 flomo，实现“碎片卡片 → AI成文”的核心链路。用户可导入 flomo 卡片，搭配主题和风格，让 AI 自动生成文章初稿，并追溯每段灵感来源。

**功能列表**

| 模块 | 需求编号 | 功能描述 | 优先级 |
|---|---|---|---|
| flomo 集成 | P2-01 | **OAuth 授权连接**：用户点击“连接 flomo”，跳转 flomo 授权页面，回调获得 access token。存储加密 token。 | P0 |
| | P2-02 | **卡片同步与浏览**：获取 flomo 的 memos 列表，支持按标签、时间范围、关键词筛选。展示卡片内容、创建时间、标签。支持分页/无限加载。 | P0 |
| | P2-03 | **选择卡片形成“灵感包”**：用户勾选若干卡片，添加到当前编辑文章的灵感区，可调整顺序。 | P0 |
| AI 成文 | P2-04 | **主题输入**：用户输入文章主题（必填）和补充说明（可选）。 | P0 |
| | P2-05 | **写作风格选择**：下拉选择内置风格（严肃深度/犀利锐评/轻松随笔/小红书种草/故事体等）。可展开预览风格描述。 | P0 |
| | P2-06 | **AI 生成文章**：点击“一鼓成文”按钮，将 选中的卡片+主题+风格 发送给后端 AI 服务，生成一篇完整文章，自动填入编辑器。 | P0 |
| | P2-07 | **卡片溯源标注**：生成文章中以温和的角标（例如 [²]）或侧边注释形式，标记哪些段落引用了哪张卡片。点击可查看卡片原文。 | P1 |
| | P2-08 | **重新生成/调整**：对生成结果不满意可单击“换个写法”，或修改主题/风格后再生成。保留生成历史。 | P1 |
| 内容连贯 | P2-09 | **手动编辑**：生成的文章可在 Phase 1 已有的编辑器中自由修改，完美衔接排版功能。 | P0 |

**关键流程：从卡片到文章**  
1. 用户在文章编辑页打开“灵感库” → 授权连接 flomo。  
2. 浏览/筛选卡片 → 勾选 5-10 条有关“早起习惯”的卡片，点击“使用选中的卡片”。  
3. 输入主题“我是如何养成早起习惯的”，风格选“轻松随笔”。  
4. 点击“一鼓成文” → AI 生成文章，包含引言、个人故事、方法、总结，并在侧边显示引用卡片编号。  
5. 用户在编辑器中修改润色 → 应用排版 → 一键复制发布。

**数据模型要点**  
- **FlomoConnection**：id, user_id, access_token(加密), flomo_user_id, connected_at  
- **InspirationPack**：id, user_id, article_id, name, card_ids (JSON数组), created_at  
- **AIGenerationHistory**：id, article_id, prompt_params(主题、风格、卡片ID列表), generated_content, created_at  
- 卡片数据在服务端不全文存储，仅缓存必要摘要以支持追溯。

**AI 服务设计**  
- API 输入：system prompt(风格定义) + user prompt(主题 + 拼接的卡片内容)  
- 模型选择：GPT-4o 或 Claude 3.5，支持流式输出，前端逐字显示。  
- 风格定义：维护一个 StyleLibrary，每个风格包含系统指令、示例、语调参数。  
- 溯源实现：Prompt 中要求模型在引用卡片内容时用特定格式 `[card_id]` 包裹，后端解析后生成角标。

**非功能性需求**  
- 卡片列表加载应快速，首次同步可异步后台完成。  
- AI 生成成功率 > 99%，生成时间普通文章 < 15 秒，流式输出让用户感受速度。  
- 卡片同步需尊重 flomo API 频率限制。

### Phase 3 Spec：妙笔生花

**版本目标**  
AI 从被动生成变为主动建议；允许用户训练个人写作风格模型；排版能力升级为杂志级；实现多平台 API 发布。

**功能列表**

| 模块 | 需求编号 | 功能描述 | 优先级 |
|---|---|---|---|
| AI 灵犀建议 | P3-01 | **智能组稿推荐**：后台每周/日扫描用户 flomo 卡片，自动聚类，推荐“你可能想写的话题”，并附上自动聚合的卡片包。展示在首页“灵感邮件”或消息中心。 | P0 |
| | P3-02 | **一键采用建议**：点击推荐话题，自动创建新文章并导入对应卡片，用户只需确认主题和风格后即可生成。 | P0 |
| 个人风格训练 | P3-03 | **风格仿写**：用户上传 3-5 篇自己满意的文章样本，系统提取语调、句式、常用词汇、段落结构等特征，生成个人专属风格模型。 | P1 |
| | P3-04 | **风格库扩充**：用户可在生成时选择“我的风格”，也可编辑、删除已训练的风格。 | P1 |
| 高级排版 | P3-05 | **新增排版组件**：分栏、首字下沉、引用卡片（带边框底色）、信息图样式列表、可定义内边距的分隔组件等。 | P0 |
| | P3-06 | **版式模板市场（初期内置）**：官方/社区提供更多精美模板，用户可预览并应用到文章。 | P1 |
| | P3-07 | **版式自定义保存**：用户可将当前文章样式保存为“我的模板”，下次直接加载。 | P1 |
| 多平台发布 | P3-08 | **平台账号绑定**：支持绑定微信公众号（需通过授权或填写 AppID/AppSecret）、小红书账号（后续如开放 API）等。 | P1 |
| | P3-09 | **文章发布/推送草稿**：一键将排版好的文章推送到微信公众号草稿箱，或调用小红书创建笔记。可填写平台特定字段（如头条标题、摘要）。 | P1 |
| | P3-10 | **发布记录**：记录发布时间、平台、文章链接，便于追踪。 | P2 |

**核心流程示例：主动建议**  
1. 用户连续记录 flomo 卡片，AI 每周分析后，推送“本周灵感快递”：话题「从远程办公反思团队信任」，附带 12 张相关卡片。  
2. 用户一键点击 → 进入新文章，卡片已填入，她用自己训练的“深度思考风”生成初稿 → 微调后应用“杂志访谈”排版模板。  
3. 最终推送到公众号草稿箱。

**技术要点**  
- 卡片扫描与聚类：可用文本嵌入（Embedding）+ 聚类算法定期处理，或直接通过 LLM 分析。  
- 风格模型微调：可采用 few-shot prompt 或 fine-tuning（如果资源允许），初期用 prompt engineering 实现。  
- 平台发布对接：公众号部分需研究其草稿接口和 access token 管理，小红书如无公开 API 则先提供图文导出规则和手动发指引。

**非功能性需求**  
- 扫描推荐不应额外消耗用户感知性能，利用空闲时段处理。  
- 高级排版组件需保证编辑流畅，移动端预览适配。

### Phase 4 Spec：文脉相通

**版本目标**  
从个人工具走向协作与生态。支持团队共享、审校，开放 API，模板市场与内容分析，构建创作者网络。

**功能列表**

| 模块 | 需求编号 | 功能描述 | 优先级 |
|---|---|---|---|
| 团队协作 | P4-01 | **团队空间**：可创建团队，邀请成员，共享文章。成员角色：拥有者、编辑、评论者、只读。 | P0 |
| | P4-02 | **实时协同编辑**：多人同时在线编辑同一篇文章，基于 OT/CRDT 算法，呈现协作光标。 | P1 |
| | P4-03 | **审阅与评论**：划词评论、文章级批注，可解决/回复评论。 | P0 |
| | P4-04 | **版本历史与回滚**：记录每次保存和发布版本，可查看差异并回滚。 | P1 |
| 开放 API | P4-05 | **公共 API**：提供 RESTful API，允许第三方通过 OAuth 访问用户授权的文章、模板、AI 生成能力。 | P1 |
| | P4-06 | **插件生态**：发布 SDK，支持编写 Obsidian、Logseq 插件，实现快速导入和一键推送排版后的文章。 | P2 |
| 模板市场 | P4-07 | **设计者入驻**：允许设计师上传排版模板，标注价格，上架审核。 | P2 |
| | P4-08 | **模板购买与使用**：用户浏览市场，一键购买并应用到自己的文章。交易分成。 | P2 |
| 内容分析 | P4-09 | **数据统计**：文章发布后，通过集成平台数据（公众号阅读量、点赞，或手动输入）提供阅读数据可视化。 | P2 |
| | P4-10 | **写作建议**：基于历史数据，AI 给出话题、标题、发布时间等优化建议。 | P2 |

**核心流程：团队协作**  
1. 团队创建“品牌月刊”空间，主笔撰写初稿 → 分享给同事，他们划词评论“这里数据需更新”。  
2. 主笔查看评论，修改后解决，其他人可实时看到更新。  
3. 定稿后应用统一模板，由管理员通过 API 一键发布到各平台。  
4. 数据回流到分析模块，指导下期内容。

**技术考量**  
- 协作编辑：采用 Yjs 或 ShareDB 实现 CRDT 协同。  
- 权限系统：RBAC 模型，团队级与文章级权限结合。  
- 开放平台：标准 OAuth 2.0 授权，API 网关限流，提供文档与 SDK。  
- 模板市场：需设计交易、审核流程，版权保护。

**非功能性需求**  
- 协作延迟 < 500ms，支持最多 10 人同时编辑。  
- API 响应时间 < 200ms (P95)，可用性 99.9%。  
- 安全：团队文档需隔离，操作全日志审计。
